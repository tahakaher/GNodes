# G-Nodes — محرر البرمجة المرئي (MVP إنتاجي)

**G-Nodes** محرر برمجيات مرئي يعمل على **لوحة عمل لا نهائية (Infinite Canvas)**؛ يبني فيه
المستخدم منطق بايثون عبر **عُقد (Nodes)** متصلة بـ **أسلاك (Wires)**، مع **مزامنة ثنائية
تفاعلية**: العُقد ↔ كود بايثون في الاتجاهين وفي الوقت الحقيقي.

---

## ✅ الميزات المكتملة

### 1) مكتبة عُقد شاملة لبناء مشاريع بايثون حقيقية
| الفئة | العُقد |
|------|--------|
| **الإدخال/الإخراج** | إدخال `input()` · طباعة `print()` · قراءة ملف · كتابة ملف |
| **المتغيرات والبيانات** | نص (String) · رقم (int/float) · منطقي (Boolean) · قائمة (List) · قاموس (Dict) · تعبير حسابي |
| **الشروط والمنطق** | شرط بسيط `if` · شرط ممتد `if/else` · عملية منطقية `and/or/not` |
| **الحلقات** | `for i in range()` · `while` · تحكّم `break/continue` |
| **المكتبات القياسية** | `random` · `time` · واجهة رسومية `tkinter` |

### 2) محرك توليد الكود (generator.js)
- **ترتيب طوبولوجي (Topological Sort)** لسلاسل التنفيذ: يبدأ من الجذور (بلا تدفّق وارد)
  ويمشي عبر منافذ `flow_out`، فيضمن عدم استخدام متغير قبل تعريفه (لا `NameError`).
- **نوعان من الأسلاك:** أسلاك **التدفّق (flow)** تحدّد التسلسل والتداخل (أجسام الحلقات/الشروط)،
  وأسلاك **القيمة (value)** تمرّر أسماء المتغيرات بين العُقد.
- كود **متوافق مع PEP 8**: مسافة بادئة 4 فراغات، `import` مجمّعة أعلى الملف بلا تكرار،
  تعقيم أسماء المتغيرات، وتغليف السلاسل بأمان.

### 3) المحلّل العكسي (parser.js) — مزامنة ثنائية
- تحرير كود بايثون في المحرّر المدمج يُعيد بناء العُقد والأسلاك حياً (Live Parsing مع debounce).
- يفهم: `import` · الإسنادات بكل الأنواع · `input/print` · `if/else` · `for/while` ·
  `break/continue` · `random.*` · `time.*` — ويستنتج التداخل من المسافة البادئة.
- **معالجة أخطاء أنيقة:** إن تعذّر فهم سطر، لا ينهار البرنامج بل يعرض **رقم السطر والسبب**
  في شريط الحالة ويُبقي آخر حالة عُقد مستقرة.

### 4) نظام حماية "صفر ثغرات" (Bulletproof)
- **توافق أنواع المنافذ:** يمنع ربط غير المتوافق (مثلاً نص ↔ منفذ تدفّق، أو flow ↔ value).
- **منع الدورات (Cycle Detection):** يمنع الحلقات الدائرية في تسلسل التنفيذ (BFS على روابط flow).
- **قيود منطقية:** لا ربط ذاتي، لا تعدّد أسلاك على منفذ إدخال، لا روابط مكرّرة.
- **تغذية راجعة بصرية:** أثناء سحب سلك، يتوهّج المنفذ الهدف **أخضر (صالح)** أو **أحمر (مرفوض)**،
  وتظهر رسائل **Toast** أنيقة عند أي رفض.

### 5) واجهة فاخرة متجاوبة (Dark Premium · Responsive · Touch)
- ثيم داكن احترافي، منافذ ملوّنة حسب نوع البيانات، أسلاك بيزييه بظلال ناعمة.
- **تجاوب كامل:** Desktop / Tablet / Mobile مع قوائم منزلقة وخلفية معتّمة.
- **دعم اللمس الكامل:** تحريك بإصبع، **تكبير بإصبعين (pinch)**، أزرار تكبير عائمة،
  وإضافة العُقد باللمس (نقرة) أو النقر المزدوج.

## 🗂️ بنية الملفات
```
index.html            الهيكل + محرّر الكود + التبويبات
css/style.css         التصميم الفاخر المتجاوب + دعم اللمس + الـ Toasts
js/nodeLibrary.js     تعريفات العُقد + الفئات + قواعد توافق المنافذ
js/graph.js           نموذج البيانات + الحماية المنطقية + منع الدورات + replaceState
js/generator.js       محرك توليد بايثون (ترتيب طوبولوجي + PEP 8)   ← جديد
js/parser.js          المحلّل العكسي (Python → Nodes)              ← جديد
js/canvas.js          محرك اللوحة: Pan/Zoom + إيماءات اللمس
js/nodes.js           عرض العُقد، الحقول المتعددة، السحب، المنافذ
js/wires.js           رسم الأسلاك + الربط + التغذية الراجعة للتوافق
js/app.js             التجميع + المزامنة الثنائية + Toasts + قوائم الجوال
```

### المعمارية
- **مصدر حقيقة واحد:** `Graph` (نمط Observer). أي تعديل → `emit` → إعادة رسم + توليد الكود.
- **حلقة المزامنة الثنائية محميّة** ضد التكرار اللانهائي عبر أعلام `updatingEditorFromGraph`
  وفحص تركيز المحرّر.

## 🌐 نقاط الدخول الوظيفية (URIs)
- `index.html` — التطبيق كامل الوظائف (صفحة واحدة، بلا خادم).

## 🎮 كيفية الاستخدام
1. اسحب عقدة من الشريط الجانبي (أو انقر عليها على الجوال) إلى اللوحة.
2. اربط منفذ **التدفّق الأبيض** (المربّع) بين العُقد لتحديد تسلسل التنفيذ.
3. اربط منافذ **القيمة الملوّنة** لتمرير المتغيرات.
4. شاهد **كود بايثون** يتولّد حياً في تبويب «كود بايثون».
5. أو حرّر الكود مباشرة → تُعاد بناء العُقد تلقائياً.

## 🚧 حدود النسخة الحالية (بوضوح)
- المحلّل العكسي يدعم **مجموعة فرعية عملية** من بايثون (كافية لعُقدنا)؛ الأكواد المعقّدة جداً
  (دوال، أصناف، تعبيرات متداخلة عميقة) تُظهر تحذيراً بدل بنائها.
- تنفيذ كود بايثون فعلياً غير مدعوم (بيئة ثابتة/Static) — التصدير فقط.

## 🧭 الخطوات المقترحة
1. حفظ/فتح مشاريع (تصدير/استيراد `.json` و`.py`).
2. Undo/Redo عبر سجل حالات.
3. توسيع المحلّل ليدعم الدوال (`def`) والأصناف.
4. تلوين صياغة (Syntax Highlighting) داخل المحرّر عبر CodeMirror.
